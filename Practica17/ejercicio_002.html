Practica 17

Estructura de los canales de contenidos

Elemento <channel>

El elemento Channel como tal hasta la fecha cuenta con todos estos elementos designados por W3C: category, cloud, copyright, description, docs, generator, image, language, lastBuildDate, link, managingEditor, pubDate, rating, skipDays, skipHours, textInput, title, ttl, webMaster. En este caso solo explicaremos los siguientes elementos: category, copyright, image, language.

  • <category>: Como ya podemos deducir por su nombre este elemento nos sirve para definir la categoría de cada canal.
  • <copyright>: Con esta etiqueta podremos definir el tipo de licencia al cual esta adscribir el canal RSS al cual corresponda este elemento.
  • <image>: Mediante esta etiqueta podremos asociar una imagen al canal del cual dependa esta etiqueta.
  • <language>: Ya por último usaremos esta etiqueta hijo en un elemento channel para designar el idioma del recurso.

Elemento <item>

Como en el caso anterior los elementos “Item” también cuentan con varios elementos hijos tales como: author, category, comments, description, enclosure, guid, link, pubDate, source, title. Dichos elementos “ítem” son a su vez hijos de los elementos Channel. En este caso puede haber varios elementos “item” dentro de un mismo elemento “Channel”. De todos los elementos con los que contamos en este caso solo hablaremos de los siguientes: autor, comments, enclosure.

  • <autor>:Si usamos esta etiqueta dentro de un elemento ítem podremos definir un autor para el mismo.
  • <comments>: Gracias a esta etiqueta podremos redactar un comentario para para cada elemento item.
  • <enclosure>: Si queremos añadir un elemento multimedia a nuestro item podremos hacerlo mediante esta etiqueta.
